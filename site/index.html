A real test gate for skills.

Two of the three checks are pure static analysis — they run anywhere Node runs, instantly, offline, on untrusted input. No Docker, no sandbox to babysit.

01 · static

lint

Does it even load? Valid frontmatter, sane name & description, and file references that actually resolve.

skillci lint ./skill
no agent required
02 · static

audit

11 safety rules for the real attack shapes: curl|sh, credential & wallet theft, secret exfiltration, prompt-injection overrides.

skillci audit ./skill
no agent required
03 · behavioral

test

Run scenario specs through a real agent CLI and assert on output & tools used. Isolation is delegated to the agent's own permissions.

skillci test ./skill
uses your installed agent

11 rules, modelled on real attacks.

Each rule is a few lines of YAML. Adding one is the easiest way to contribute — write the pattern, add a fixture line, done.

// transparent on purpose

Boring, legible scoring.

No black-box “AI trust score.” Start at 100 and subtract. A skill is VERIFIED only with zero errors and every test that ran passing. Skipped tests never count against you.

score = 100
  −25 per error
  −8  per warning
  −2  per info
  −15 per failed test

verified = errors == 0 && failures == 0

Wire the badge into any README the moment a skill goes green:

$ skillci badge ./skill

Three ways in. Each one tiny.

skillci is deliberately modular so the barrier to a first PR is a markdown file or less.

01

Add a safety rule

Spotted a risky pattern? Six lines of YAML in rules/core.yml plus a fixture line. That’s the whole PR.

02

Add an agent adapter

Make skillci test through Codex, Cursor, Gemini CLI… implement one interface in one file. No changes anywhere else.
